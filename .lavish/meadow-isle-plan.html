Meadow Isle — Development Plan

REVISED · 2026-07-18 · research integrated design authority: docs/islands/meadow-isle.md

Turn the first creature island from a design doc into a playable, curriculum-true Standard-1 experience: walkable island, catchable creatures, and a question engine that asks like a real SJKC worksheet — counting sheep, reading clocks, counting coins.

Design authority: docs/islands/meadow-isle.md · Curriculum: docs/curriculum/standard-1-sjkc-math.md + standard-1-question-style.md · World: docs/world-map-design.md

Where we stand (verified against the repo, 2026-07-18)

✅ M1 region graph

game/assets/src/world/regions/ now contains Harbor and all eight Meadow regions, named gateways, camera follow, and region travel. The M1 implementation is in final review.

✅ Battle + shop loops

BattleScreen.ts, QuestionView.ts, shared/battle-rules.ts, shared/shop-rules.ts — turn-based battle and a shop exist.

✅ Two safe art paths

remote-art.ts streams licensed pack art from private R2, while tools/generate-creature.mjs can produce original PokeMath evolution strips. Neither art source becomes save identity.

⚠️ Question engine is Year-4 shaped

shared/question-engine.ts = numeric answer + makeChoices() near-misses. Std 1 needs visual formats, non-numeric answers, TP levels, profile gating. This is the long pole.

⚠️ Foundation migration owed

Stats still live on creatures, saves have no persistent region, collection storage, Field Guide, or player XP, and species IDs are not separated from art references. M1.5 resolves these together.

✅ Curriculum ground truth

Two research docs pin scope + formats with real-paper evidence and a validated schema (§E). The generator prompt is mostly written already.

✅ Reward direction

Player-owned XP, a compact victory result, calm normal captures, collection as a first-class reward, and Unique-only flee pressure are now locked. No streak loss, answer timers, or idle punishment.

Revised milestones — foundations first, then vertical slices

M1 · finishing

Sail there & walk it

Keep the current region graph, named gateways, camera follow, banners, ferry confirmation, safe arrivals, and return route. M1 remains session-only.

Playable exit: walk every Meadow region and sail home; region tests and typecheck pass.

M1.5 · new gate

Save, progression & collection foundation

Land save v2 before encounters: player-owned { level, totalXp }, persistent regionId + x/y, semantic speciesId, replaceable artRef, stable creatureId, starterCreatureId, ownedCreatures, active teamIds[≤6], Field Guide seen/caught/variant state, badges, and curriculum profile.

Playable exit: v1 saves migrate without lost progress; a seventh capture enters storage; team selection remains valid.

M2A

One satisfying encounter loop

Ship one Meadow area with a small real roster, a mechanically checked ~20-item Std-1 numeric bank, calm ordinary capture, gentle defeat, immediate answer feedback, deterministic player XP, persistent XP bar, and compact victory result. Capture never awards full kill XP when it bypasses the remaining questions.

Playable exit: 2–4 battles produce a clear 10–15 minute adventure and every reward displayed equals the saved result.

M2B

Expand encounters & collection

Expand the stable species registry and habitat tables across Meadow. Licensed R2 art and original generated art are interchangeable behind artRef; pack paths never enter saves. Add Field Guide UI and a minimal Harbor Sanctuary for storage/team changes. Common, uncommon, and rare captures remain calm and untimed.

Playable exit: discover, catch, store, inspect, and field creatures from every ordinary rarity without losing one at 6/6.

M3 · long pole

Std-1 question schema

Add schema-v2 questions with format_type, tp_level, presentation, answer_form, misconception-driven distractors, bilingual values, and profile gating. Preserve legacy schema-v1 behavior byte-for-byte through adapters and golden tests.

Playable exit: numeral, objective circle, and ordering forms work; legacy battle and shop remain unchanged.

M4

Validated batch generator

Generate offline by grade/topic/TP/profile, mechanically reject curriculum violations and wrong answers, run an adversarial verify pass, then human-review the first 200 and sample later batches. Serve only approved banks; no generation runs during a child's battle.

Playable exit: every island encounter draws from an approved, rollback-safe Std-1 bank.

M5

Landmarks, topic arcs & session payoffs

Build the shared FigureView kit (ten-frame, clock, coins, objects first), then wrap landmarks in short goals that repair a shortcut, help an NPC, reveal a habitat clue, or visibly change the region. Add selected mini-guardians and evolution beats across levels 1–25.

Playable exit: each 10–15 minute session can resolve one chosen intention and reveal a non-urgent next possibility.

M6

Unique hunt, guardian, badge & evolution

Author a Unique research trail with persistent clues. Only Unique rarity gets telegraphed action-count flee pressure and a larger trust meter; question thinking time remains unlimited and escape never erases trail progress. Finish with the fixed Cloud-Maned Horse guardian slate, Meadow Badge, and guaranteed starter evolution.

Playable exit: Meadow is completable end-to-end and the Unique hunt always offers a fair second chance.

M7

Observed tuning & healthy return

Tune the already-shipped XP constants, level gaps, effort, catch rates, variants, and question-rank calibration from observed battles. Add telemetry for correctness, hints, abandonment, delayed review, collection variety, and healthy stopping. Only after enough content exists, add the optional three-separated-day expedition journal and 20–30% interleaved review—never a breakable streak.

Playable exit: progression feels fair from levels 1–30 and returning supports learning without guilt or pressure.

Critical path: M1 → M1.5 → M2A → M2B → M3 → M4. M5 can overlap late M4. M6 needs M2B + M4 + M5. M7 tunes real behavior; it does not postpone the XP system itself.

Locked implementation decisions

D1 · Question generation timing

A · Offline batch → validate → D1 recommended

Generate in bulk via tools/, gate hard, ship approved items. Zero latency, cost-controlled, kid-safe by construction. Matches ROADMAP's "validation gate" instinct.

B · Generate at serve time

Infinite variety, but latency in a kid's battle, per-request cost, and validation must run inline on theWorker hot path.

C · Hybrid

Batch seed + background top-up when a topic×TP cell runs low. Best long-term; more moving parts now.

D2 · Save and collection foundation

Land save v2 as M1.5, before encounters locked

Progression, persistent location, creature identity, collection storage, Field Guide state, badges, and curriculum profile migrate together so later milestones build on one final ownership model.

Save migration plan (v1 → v2)

Honest starting point: validateSaveState today rejects anything but version === 1, and SaveState has no player stats, badges, or curriculum profile. "Versioning exists" means a version field, not a migrator.

  • Upgrader, not rewriter. New migrateSave(raw) in shared/: accepts only well-formed v1 (version: 1 required — absent stays invalid, exactly as today), then validates-as-v1 → migrates → validates-as-v2. One-way, tested per version pair.
  • Strict decode, no pass-through. validateSaveState is a trust boundary: decode only known v1 fields and construct a fresh v2 object from them — arbitrary unknown fields are dropped at the boundary, never carried into the new schema. Every known v1 field has a defined home: team (names, colors, boss flags), money, bag, savedAt carry over verbatim; per-creature stat fields become derived-from-player going forward (world-map model) and are retained read-only for one version, then removed in v3.
  • Identity is independent of art. Each creature gains semantic speciesId, stage, variant, and stable creatureId; species data owns a replaceable artRef. The save gains starterCreatureId. Legacy creatures map to reserved legacy/<name> species and stay playable forever.
  • Collection added in v2. ownedCreatures stores every creature; teamIds references at most six; Field Guide state records unknown/seen/caught species and discovered variants. A seventh capture enters storage instead of being rejected.
  • Location added in v2. location: { regionId, x, y } — v1 saves migrate to the Harbor Town spawn. On every load the saved tile is validated walkable; if a layout edit made it solid, spawn falls back to that region's named safe gateway.
  • Concrete XP conversion. New player total XP preserves the highest creature's old level and fractional progress, then uses the approved variable player-level curve. No child silently loses ground and old flat-20 XP is never treated as the new threshold.
  • Rollout order. Worker deploys the accept-and-migrate reader first; the game client ships v2 writes only after. D1 stores saves as JSON payloads keyed per account, so migration happens at the read/write boundary — no DB schema change.
  • Acceptance tests. Fixture v1 saves (fresh createNewGame() + a mid-game example) must migrate, re-validate, round-trip, and boot into a playable battle. A corrupted or future-version save still fails cleanly.
Do not split this across repeated migrations.

Shipping player XP first and storage later would churn the same trust boundary and still leave capture built on a temporary model.

D3 · Answer-form scope for first playable

A · numeral + circle (objective) + ordering first recommended

Covers most of §A's high-frequency formats (count-write, compare, order-sequence, fill-blank, true-false) with UI we already half-have. match/color/tick follow in M5; drawing/trace-write are cut from the island entirely (see Risks).

B · All 10 answer forms at once

Tracing/drawing and colouring need bespoke touch UI; bundling them delays everything behind the hardest 20%.

D4 · Guardian question slate

A · Fixed authored slate v1 recommended

The guardian is a set piece — one hand-checked question per topic (count, +−, clock, money, pictograph). Deterministic, reviewable, no generator dependency for the island's one critical battle.

B · Mastery-weighted sampling

Targets each child's weakest topics — great pedagogy, but needs per-topic mastery tracking that doesn't exist yet. A later-island upgrade.

D5 · Capture pressure by rarity

Unique rarity only locked by Season

Common, uncommon, and rare encounters use the calm ordinary capture flow with no flee clock and no timed mathematics. Only an authored Unique hunt may use telegraphed remaining actions and a larger trust meter. Thinking time per question is always unlimited; a fleeing Unique preserves its trail, silhouette, habitat clue, and next chance.

D6 · Healthy retention boundary

Satisfying sessions before return mechanics locked

Build clear battle payoff, collection, short goals, shortcuts, and topic arcs first. The three-day expedition journal waits until M7 and never resets progress. No hard streaks, answer-speed XP, daily obligation, idle pet punishment, public leaderboard, gacha, or extra currencies.

Risks

HighGenerator leaks out-of-level content. The research docs' own verify pass dropped real exam items for being multi-step — prompt wording alone won't hold the line.
Recommended Three-layer gate in M4, all in tools/validate-question.ts: (1) mechanical — parse the expression, hard-reject × ÷, any operand/result > 100, multi-step chains; money parser rejects > RM10 and mixed coin+note exchange; validator independently re-derives the stated answer and rejects mismatches; (2) adversarial — a second LLM call runs the corpus' own checklist (authentic / on-level / correctly-answered) and rejects, never edits; (3) human sample — you review the first 200 items 100%, then a 5% random sample per batch.
MediumFigure-render scope creep. 13 presentation types are each bespoke UI.
Recommended One FigureView kit, config not custom UI. Every figure is a small declarative spec (rows of icons, labels, highlights) rendered by shared primitives — ten-frame, clock, coins, objects, pictograph are the same grid-with-slots renderer with different skins. Ship the 4 highest-frequency types (ten-frame, clock, coins, objects) in M5; any landmark whose figure hasn't landed falls back to "prose + world sprites" by rule, not by schedule slip. drawing/trace-write answer forms are cut from Meadow Isle entirely — "set the clock hands" becomes tap-to-set (an objective interaction), no freehand input.
LowBilingual volume. Every string needs zh + en gloss, and creature names are still placeholders.
Recommended Glossary-first. Create shared/strings.ts + docs/i18n-glossary.md in M2 holding the frozen curriculum terminology (数值, 数位, 整点, 一刻…) so generator prompts and UI reuse identical wording. Creature names: one LLM naming pass (zh-first, kid-pronounceable) reviewed by you in a single sitting — 12 families, one batch, done. English gloss stays secondary exactly as QuestionView already renders it.
MediumSave migration. validateSaveState currently rejects everything except version === 1 and there is no migrator — D2 without one silently invalidates existing saves.
Recommended Land M1.5 as its own migration sequence before encounters: migrateSave() upgrader (v1 → migrate → validate-as-v2), explicit preservation rules, player progression, location, semantic species identity, owned storage/team references, and Field Guide state. Worker deploys the migrating reader before the client ships v2 writes; fixture saves (fresh + mid-game) remain permanent regression tests.
MediumM3 regression bar. The versioned engine must not break the legacy numeric path that battle & shop still run on.
Recommended Golden-file equivalence test. Snapshot today's engine output for every SAMPLE_BANK question (turns, choices, judge results) into a fixture; M3's engine must reproduce it byte-for-byte through the legacy path. Add it to the existing test run so any PR that changes v1 behaviour fails immediately. New V2 forms stay additive behind the profile flag; if the bar can't be met, M3 slips rather than v1 breaking.
